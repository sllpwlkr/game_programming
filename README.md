# 3D Puzzle Game (Unity) — Отчёт о проекте
 
## Описание
 
Игрок управляет одним или несколькими кубами, перемещая их по игровому полю с целью достижения заданных условий уровня. Основная идея заключается в управлении объектами различного размера и цвета, их разделении и объединении, а также взаимодействии с элементами уровня: стенами, проходами, переключателями гравитации и целевыми зонами.

## Игровые механики
 
### Перемещение и выбор куба
 
Управление осуществляется клавишами WASD. На полу все кубы перемещаются одновременно, тогда как на стене управляется только один - активный куб. Перемещение происходит дискретно по клеткам сетки. Переключение между кубами выполняется клавишей **Tab**, а активный куб визуально выделяется на фоне остальных.
 
### Деление и объединение
 
Куб можно разделить на два меньших нажатием **Space**. Деление доступно только на полу, только если куб не минимального размера и рядом есть свободные клетки.

Цвета новых кубов определяются по таблице:
- красный → синий + жёлтый
- синий → красный + жёлтый
- жёлтый → красный + синий
 
Объединение выполняется клавишей **E** и возможно для двух соседних кубов одинакового размера с допустимой комбинацией цветов. Кубы одинакового цвета и размера также могут объединиться. В результате образуется куб большего размера, чей цвет определяется сочетанием исходных.
 
В игре поддерживаются три размера кубов: большой (Large), средний (Medium) и маленький (Small). Размер влияет на возможность деления, прохождение через узкие проходы и соответствие целевым условиям уровня.
 
Цвет куба определяет, через какие цветные препятсвия(стены/полы) он может проходить, и учитывается при выполнении целей.
 
### Проходы (ворота)
 
Фиолетовые ворота пропускают только маленькие кубы, оранжевые — только средние.
 
### Система гравитации и передвижение по стенам
 
В игре реализовано переключение активной поверхности между тремя состояниями: пол, левая стена и правая стена. Переключение происходит через специальные переключатели, которые активируются только при наличии ровно одного куба и соответствии текущего режима условиям переключателя.
 
При активации гравитации куб перемещается на соответствующую стену, управление адаптируется под новую плоскость, а камера автоматически меняет положение.
 
### Камера
 
Поддерживаются четыре режима обзора: вид на пол, вид на левую стену, вид на правую стену и автоматический режим. Переключение выполняется клавишами **1**, **2**, **3** и **0** соответственно.
 
### Целевые зоны и завершение уровня
 
Игрок должен разместить кубы в целевых ячейках. Каждая цель может требовать куб определённого цвета, определённого размера или принять любой куб. После выполнения цели куб фиксируется и меняет визуальное состояние.
 
Уровень считается завершённым, когда все цели выполнены. После этого отображается экран победы.

## Управление
 
| Клавиша | Действие |
|---|---|
| WASD | Перемещение |
| Tab | Выбор куба |
| Space | Разделение |
| E | Объединение |
| R | Перезапуск уровня |
| 0 / 1 / 2 / 3  | Переключение камеры |
 
 
## Интерфейс
 
Реализованы стартовое меню, окно с правилами (на нескольких страницах), экран победы, кнопка перезапуска уровня и система отображения текущих целей в процессе игры.
 
 
## Архитектура проекта
 
Проект разбит на следующие основные компоненты:
 
| Компонент | Назначение |
|---|---|
| `GameManager` | Управление игровым процессом |
| `CubeUnit` | Логика и визуал кубов |
| `CubeActionSystem` | Деление и объединение |
| `LevelQuery` | Проверки столкновений и окружения |
| `GoalZone` | Целевые зоны |
| `GravitySystem / GravitySwitch` | Система гравитации |
| `CameraViewController` | Управление камерой |
| `LevelUIController` | Пользовательский интерфейс |
